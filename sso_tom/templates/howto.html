How to use the SSO Alert System

Introduction

This page provides basic instructions on how to use the SSO Alert System.

This system provides the ability to automatically submit observations to the ANU 2.3m as well as other facilities.

By the end of this guide, you should be able to submit an observation to the ANU 2.3m via the SSO Alert System

In order to use this service you should be familiar with the "Applying for time on the ANU 2.3-metre telescope" page.

This instance of the SSO Alert System is deployed by the Research School of Astronomy and Astrophysics at the Australian National University

For additional assistance or enquires please contact: sso-alert-system@mso.anu.edu.au

Registration

Register here with the email address used with LENS. The same page can be used to log into the SSO Alert System when registered.

Your account will need to be approved before you are able to access the system.

(back to top)

Connecting to LENS

In order for the SSO Alert System to submit an observation to the ANU 2.3m on your behalf, you need to add the "SSO Alert System" as a co-investigator to your LENS proposal.

The "SSO Alert System" should be granted observations access.

(back to top)

Creating an observation template

If you are planning to submit multiple observations with similar configurations, the first step is to create an observation template.

Once this template is saved, the information in the template can be easily loaded for each new observation

(back to top)

Creating a target

Targets can be created by multiple methods: either manually or automatically by connecting to a broker stream.

(back to top)

Manually creating a target

On the targets page, the "Create Targets" dropdown menu contains different options. Target details can be filled in here.

(back to top)

Receiving alerts from brokers.

Targets can be automatically created by listening to different topics from Fink broker here. Fink generates a stream of targets based on each topic which will be automatically added to your account. These targets can be automatically observed without any human intervention by selecting automatic observability. More details below.

(back to top)

Submitting an observation

An observation can be manually or automatically triggered.

(back to top)

Manual observation

From your list of targets, an observation can be manually submitted by selecting a target from the targets page, clicking the observe tab and then either using an existing observation template or manually inputting the configuration desired for the observation.

(back to top)

Automatic observation

When subscribing to a broker stream, if "Automatic observability" is selected, an existing observation template is automatically applied to your saved targets and observations sent to the selected facility. (This feature is not recommended for broker streams with large numbers of candidates as your observation time may be used up quickly.)

(back to top)

Updating an observation status

Observation statuses are automatically updated every 15 minutes. However, they can also be manually updated faster by clicking "Update Observations Status"

(back to top)

Cancelling an observation

Observations can be cancelled by clicking on "View" for an observation and selecting "Cancel Observation"

(back to top)

Chaining observations

Observations can be "chained", that is, sending observations to multiple facilities in a row.

For example, first send the target to a photometric facility and if a source is detected, send the target to a spectroscopic facility.

This feature has been implemented, however, there is only one facility currently available.

(back to top)
